# 📱 Google Play Store Analytics Dashboard

## 📌 Project Overview

This project analyzes Google Play Store application data to uncover insights related to app performance, user engagement, ratings, reviews, installs, and category trends. Using Python and Power BI, the project transforms raw app data into actionable business insights through data cleaning, exploratory data analysis (EDA), and interactive dashboard development.

The project demonstrates a complete Data Analytics workflow:

**Data Collection → Data Cleaning → Exploratory Data Analysis (EDA) → Dashboard Development → Business Insights**

---

## 🎯 Business Problem

The Google Play Store hosts thousands of applications across multiple categories. Understanding the factors that influence app success can help businesses, developers, and marketers make better decisions.

This project aims to answer key business questions:

- Which app categories receive the highest installs?
- How do ratings impact app popularity?
- Which categories generate the most user engagement?
- What characteristics do top-performing apps share?
- How do reviews and ratings influence installs?

# 📊 Key Insights

### Category Performance

- Identified the top-performing app categories based on installs and ratings.
- Discovered significant differences in performance across app categories.

### Ratings & Installs

- Higher-rated applications generally showed stronger install performance.
- Top-rated apps contributed a substantial share of total installs.

### User Engagement

- Apps with higher review counts typically demonstrated stronger user engagement.
- Review volume acted as an indicator of app popularity.

### Free vs Paid Apps

- Free applications dominated total installs compared to paid applications.
- Paid applications represented a smaller but valuable market segment.
